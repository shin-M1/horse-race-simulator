# 競馬レースシミュレーター

Streamlitで操作できる競馬レースシミュレーションアプリです。ブラウザ上でレース条件と出走馬を入力し、近走5走の分析、脚質・能力値推定、レース展開予測、100m区間ごとのシミュレーション、GIF/MP4動画生成まで実行します。

## セットアップ

```bash
pip install -r requirements.txt
```

投稿用MP4出力には `imageio` と `imageio-ffmpeg` を利用します。Blenderが利用できる環境ではBlenderによる高品質3Dレンダリングを試み、利用できない場合は簡易3D風CG動画にフォールバックします。

## 実行方法

```bash
streamlit run app.py
```

ローカル実行時の `http://localhost:8501` や `http://127.0.0.1:8501` は自分のPC専用URLです。そのまま友人へ送っても相手の環境からはアクセスできません。

## 入力項目

- 競馬場: 東京、中山、京都、阪神、中京、札幌、函館、福島、新潟、小倉
- 芝/ダート
- 距離
- 回り: 右、左
- 天候: 晴、曇、雨、雪
- 馬場状態: 良、稍重、重、不良
- 開催日数: 1日目～12日目
- 使用コース: A、B、C、D
- トラックバイアス: 標準、前残り、差し有利、内有利、外差し有利、内前有利、外伸び
- 出走頭数: 2〜18頭
- 出走馬: 馬名、枠順、馬番、斤量、騎手
- 軽量モード: 公開環境向けにMP4生成を省略し、Plotly表示と短い計算設定を使用

## 出力

- 着順表
- 各馬の能力分析表
- レース展開予測: 先頭候補、中団候補、後方候補の3区分
- レース動画 GIF または3D Plotly HTML
- 投稿用MP4: YouTube横長16:9、TikTok縦長9:16を選択可能
- 予想モードON時のMonte Carlo予想ランキング、勝率、連対率、複勝率、AI予想印
- レース強度、ELO、相対上り、コース適性、展開適性、騎手・トラックバイアス補正
- 予想印はscore順に上位5頭だけ、◎、○、▲、△、☆を1頭ずつ付与します。
- ffmpeg利用可能時のみMP4
- `outputs/` 配下のCSVと動画ファイル

## 予想・実結果・成績分析

画面は「予想」「実結果入力」「成績分析」「YouTube出力」の4タブに分かれています。

- 予想: レース名と開催日からrace_id、出馬表、レース条件を取得し、編集後に予想を実行します。
- 実結果入力: レース後に「netkeibaから実結果を取得」を明示的に押した場合だけ結果ページへアクセスします。
- 成績分析: 保存済み予想と実結果の比較ログから、◎勝率・◎複勝率・印の複勝圏率などを集計します。

予想時には実結果を取得しません。結果未公開または結果テーブルが空の場合も予想や出馬表取得は停止せず、レース後の再取得を案内します。予想時点の `actual_result` は `None` のまま `outputs/prediction_logs/` に保存され、後日の結果はrace_idで予想ログと結び付けて `outputs/evaluation_logs/` に保存されます。

netkeibaのHTML構造変更やアクセス制限で取得できない場合は、各タブのCSVアップロードまたは手入力を利用できます。取得失敗時に架空のrace_id、出走馬、レース結果を生成することはありません。

### 予想ログと評価ログ

- 予想完了時に `outputs/prediction_logs/prediction_YYYYMMDD_HHMMSS.json` と同名CSVを保存します。
- race timelineは `outputs/prediction_logs/timelines/` に分離して保存します。
- 実結果との比較後に `outputs/evaluation_logs/evaluation_YYYYMMDD_HHMMSS.json` と同名CSVを保存します。
- 評価ログには印別着順、1着・複勝圏・3連複候補の的中状況、勝因・敗因レビューを含みます。
- 実結果は馬番のカンマ区切りでも入力できます。

同じrace_id・レース名・開催日の予想ログ、または同じ予想ログに紐づく評価ログが既にある場合、既定では追加保存しません。画面で「上書き保存」または「別名で保存」を選ぶこともできます。

成績分析タブの「ログ管理」では、予想ログ・評価ログの一覧表示、重複検出、選択削除ができます。削除には確認チェックが必須です。予想ログを削除する際は、紐づく評価ログも同時に削除できます。JSONを削除すると同名CSVも削除されます。

### 予想エンジン

予想モードでは次のエンジンを選択できます。

- `rule_based`: 既存のMonte Carlo集計とルールベース評価を利用します。
- `optimized_weights`: `outputs/model_weights.json` の最適化済み重みを利用します。ファイルがない場合は既定重みを使います。
- `ml_model`: `outputs/models/top3_model.pkl` と任意の `win_model.pkl` を利用します。top3モデルがない場合は `rule_based` へ戻ります。

成績分析タブでは、評価ログから学習データセットを作り、ランダム探索による重み最適化とLogistic Regression／Random Forestによるモデル学習を実行できます。scikit-learnが利用できない環境では、最小限のNumPyロジスティックモデルへフォールバックします。

評価ログが20レース未満の重み最適化、50レース未満のML学習は結果が不安定になりやすいため、画面上に警告を表示します。

### 予想レポート

予想実行後に、AI予想印、全頭短評、展開予測、代表シミュレーション、リスク要素をまとめた「予想レポート」を表示します。YouTube台本、ブログ本文、SNS投稿文はテキスト欄からコピーできます。

「予想レポートを保存」を押すと、`outputs/reports/` に同名のMarkdownとJSONを保存します。JSONには元の予想ログパス、生成内容、生成時刻を記録します。

### YouTube出力

「YouTube出力」タブでは、現在のシミュレーション結果または保存済み予想ログから、YouTube用サムネイルと予想動画を生成できます。

- サムネイル: `outputs/thumbnails/thumbnail_YYYYMMDD_HHMMSS.png`
- YouTube予想動画: `outputs/youtube_videos/youtube_prediction_YYYYMMDD_HHMMSS.mp4`
- 同レース過去傾向: `outputs/race_trends/same_race_history_*.json`
- AI読み上げ音声: `outputs/narration/`
- 一時画像: `outputs/youtube_videos/tmp/`

動画構成は固定で、`同レースの過去傾向`、`全頭診断`、`シミュレーション動画の再生`、`注目馬の紹介と根拠説明` の順に生成します。過去10年傾向は、レース名と開催日から過去同名レースの結果を取得できた場合のみ分析し、取得できない年のデータをダミー生成することはありません。過去傾向データが不足する場合は、競馬場、距離、芝/ダート、回り、トラックバイアス、脚質分布から簡易コメントを作ります。

過去傾向では、枠順、馬番、脚質、人気、上り、騎手継続、血統系統を集計し、`race_trend_score` と `trend_match_comment` としてAI予想ランキング、各馬分析、全頭短評に反映します。全頭診断は馬番順に表示し、印列は出さず、長い馬名でも崩れにくいよう6頭単位でページ分割します。注目馬紹介は `☆`、`△`、`▲`、`○`、`◎` の順に表示し、短評とは別に詳細根拠と個別リスクを表示します。

AI読み上げをONにすると、過去傾向、全頭診断、注目馬紹介の台本からナレーション長を推定し、各セクションの表示秒数を固定値ではなく読み上げ量に合わせます。BGM、シミュレーション用BGM、スタート効果音はアップロードできます。音声合成や音声結合に失敗した場合でも、動画生成自体は継続します。

サムネイルとYouTube動画は、実際の競馬映像、JRA映像、netkeiba画像などの外部素材を使わず、自作の図形・文字・背景だけで生成します。サムネイルでは予想印や馬名を出さず、レース名とAI予想企画で訴求します。実在馬名やレース名は動画内テキストとして使用します。

### 原因分析

評価ログにはペース、トラックバイアス、上り性能、能力、斤量、騎手、脚質、レースレベル、相手関係などの原因タグと改善提案を保存します。成績分析タブではタグの回数・割合、頻出改善提案、外れレースごとの予想印・実結果・各馬レビューを確認できます。古い評価ログも、保存済みの予想表と実結果があれば画面表示時に分析できます。

## 投稿用MP4

サイドバーの「アニメーション形式」で「投稿用MP4」を選ぶと、`outputs/race_movie.mp4` を生成します。

- YouTube向け: 1920x1080、16:9
- TikTok向け: 1080x1920、9:16
- 動画の長さ: 30秒、60秒、90秒
- 表示内容: コース名、距離、馬場状態、残り距離、現在順位上位5頭、ゴール後の最終順位、AI予想印
- 馬の表示は枠色を反映した馬番入り丸印です。馬名や騎手名、実在映像素材は表示しません。
- アニメーション位置は `simulator.py` が出力する `race_timeline` を利用し、動画側で独自に位置取りを作りません。

映像はすべて本アプリ内で生成するシミュレーションCGです。実際の競馬映像、JRA等の映像素材、外部動画素材は使用していません。

## 友人に共有する方法

友人や他の人に使ってもらうには、Streamlit Community Cloud、Render、Railway、Docker + VPS などへデプロイしてください。まずは Streamlit Community Cloud に対応しやすい構成として、`requirements.txt`、`packages.txt`、`.streamlit/config.toml` を用意しています。

### Streamlit Community Cloud

1. GitHubにこのプロジェクトをpushします。
2. Streamlit Community Cloudにログインします。
3. `New app` を選択します。
4. GitHubリポジトリを選択します。
5. `Main file path` に `app.py` を指定します。
6. `Deploy` を実行します。
7. 発行された公開URLを友人に共有します。

BlenderはStreamlit Community Cloudでは利用できない可能性が高いため、Cloud上ではPillow/imageioやPlotlyによるフォールバック表示を前提にしてください。

### Render

`render.yaml` を同梱しています。GitHubへpushした後、Renderで `New +` → `Blueprint` を選び、このリポジトリを指定してください。手動でWeb Serviceを作る場合の起動コマンドは次のとおりです。

```bash
streamlit run app.py --server.port $PORT --server.address 0.0.0.0
```

Render環境では `RENDER=true` により軽量モードが標準で有効になります。デプロイ完了後にRenderが発行する公開URLを共有してください。

## 既存の直近5走取得機能との連携

サイドバーの「既存の直近5走取得モジュール」に、既存モジュール名とファクトリ関数名を入力できます。未入力の場合はnetkeibaから実データを取得します。取得できない馬名ではシミュレーションを停止します。

既存モジュールは以下のような provider を返してください。

```python
class MyProvider:
    def get_recent_results(self, horse_name: str, limit: int = 5):
        return [
            {
                "race_name": "前走",
                "distance": 1800,
                "surface": "芝",
                "track_condition": "良",
                "finish_position": 2,
                "margin": 0.2,
                "passing_order": "3-3-2",
                "final_3f": 35.4,
                "field_size": 16,
            }
        ]


def get_provider():
    return MyProvider()
```

## 注意点

- 実在しない馬名、horse_idが取得できない馬名、近走テーブルが取得できない馬名では、分析やシミュレーションは実行されません。
- サイドバーの「デバッグ情報を表示」をONにすると、入力馬名、取得馬名、horse_id、取得URL、raw DataFrame、整形後recent_racesを確認できます。
- 過去レース評価には、レースクラス、人気、頭数、着差から推定した簡易的な相手関係補正を含めています。相手馬の詳細データを捏造することはありません。
- ELOは同じ`race_id`で実際の出走馬と着順を確認できたレースだけ更新し、対戦情報がなければ1500の中立値を使います。
- 騎手データが取得・入力されていない場合、騎手補正は50の中立値です。
- トラックバイアスはユーザー選択による簡易補正で、実際の当日傾向を自動取得するものではありません。
- 脚質判定は通過順位を出走頭数で割った位置比率を利用します。頭数欠損時は通過順から推定し、デバッグモードに警告を表示します。
- `actual_running_style` はレース開始前に確率profileから決定し、レース中や着順確定後に位置・順位から再判定しません。
- 本アプリの予想は、取得データとシミュレーション条件に基づく参考情報であり、馬券的中を保証するものではありません。
- 開催日数と使用コースは、内外・先行差しの簡易的なコースバイアス補正として利用しています。
- このコースバイアスは実際の馬場差やトラックバイアスを完全に再現するものではありません。
- 枠順補正は微調整にとどめ、枠順だけで勝敗が大きく変わらないようにしています。
- 投稿用MP4は実際の競馬映像ではなく、オリジナルのシミュレーションCGです。
- Blenderがある場合は高品質3Dレンダリングを試し、ない場合は簡易3D風レンダリングにフォールバックします。
- スクレイピング先サイトが外部アクセス制限をしている場合、Cloud環境ではnetkeibaのデータ取得に失敗する可能性があります。その場合はCSVアップロードまたは手入力を利用してください。
- 現在の能力推定はルールベースです。
- 動画生成は形式によりPlotly、matplotlib、Pillow/imageio、Blenderを使い分けます。出走頭数や動画秒数が長いほど処理時間が増えます。
- 3D表示にはPlotlyを使用し、`outputs/race_3d.html` を生成します。
- MP4はffmpegが利用できる環境でのみ生成されます。
